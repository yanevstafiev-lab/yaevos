# os-bot — этап 1 «Личной ОС»

Минимальный бот с двумя ручными командами (без расписания — оно будет на этапе 2):

- `/checkin` — отправляет утреннее сообщение в рабочий чат.
- `/rate <курс>` — рассылает курс всем партнёрам из фиксированного списка.
- `/whoami` — служебная команда: любой человек, написав её боту, узнаёт свой `user_id` и `chat_id`. Нужна, чтобы собрать ID для `ADMIN_IDS`, `WORK_CHAT_ID` и `PARTNER_CHAT_IDS`.

## Важно про Telegram

Бот не может первым написать пользователю, который ни разу не писал боту. Поэтому:

- Ваш рабочий чат (для `/checkin`) — либо личный чат с ботом (напишите ему что угодно один раз), либо групповой чат, куда бота один раз добавили.
- Каждый партнёр (для `/rate`) должен один раз написать боту `/whoami` — так вы получите его `chat_id` и сможете добавить в `PARTNER_CHAT_IDS`. После этого рассылка идёт без повторных действий с его стороны.

## Локальный запуск (проверить перед деплоем)

```bash
npm install
cp .env.example .env
# впишите BOT_TOKEN и остальные переменные в .env
npm start
```

Напишите боту `/whoami`, чтобы получить свой `user_id`, впишите его в `ADMIN_IDS` в `.env`, перезапустите — и `/checkin` с `/rate` станут доступны вам.

## Деплой на Railway

1. Залейте эту папку в свой GitHub-репозиторий (Railway разворачивает из git).
2. На [railway.com](https://railway.com) → **New Project** → **Deploy from GitHub repo** → выберите репозиторий.
3. В разделе **Variables** проекта задайте те же переменные, что в `.env.example` (`BOT_TOKEN`, `ADMIN_IDS`, `WORK_CHAT_ID`, `PARTNER_CHAT_IDS`, при желании `MORNING_TEMPLATE`/`RATE_TEMPLATE`). Файл `.env` в репозиторий не попадает (см. `.gitignore`) — переменные задаются только в интерфейсе Railway.
4. Railway сам выполнит `npm install` и `npm start` (команда `start` уже прописана в `package.json`).
5. Проверьте логи деплоя — должна появиться строка `Бот запущен (long polling).`
6. Напишите боту в Telegram `/help`, чтобы убедиться, что он отвечает.
